  <action>
Create `scripts/local_parse.sh` with the following structure:

1. **Header and sourcing:**
   - Shebang `#!/usr/bin/env bash`
   - Comment header: "local_parse.sh -- Claude Code JSONL session log parser for claudux"
   - Comment: "JSONL schema is undocumented and may change. Fields verified 2026-03-10."
   - Source helpers.sh: `CURRENT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` then `source "$CURRENT_DIR/helpers.sh"`

2. **Plan limits table: `_get_plan_limits()`**
   - Parameters: `$1` = plan type (free, pro, max_5x, max_20x)
   - Define approximate token limits per plan tier per week (these are rough estimates, documented as approximate):
     ```
     free:    weekly_limit=500000    monthly_limit=2000000
     pro:     weekly_limit=5000000   monthly_limit=20000000
     max_5x:  weekly_limit=25000000  monthly_limit=100000000
     max_20x: weekly_limit=100000000 monthly_limit=400000000
     ```
   - Echo as space-separated: `echo "$weekly_limit $monthly_limit"`
   - Add comment: "# Approximate limits -- actual values may differ. Update as Anthropic publishes official numbers."
   - Default to max_5x if plan type unknown

3. **Helper: `_get_cutoff_epoch()`** -- Cross-platform epoch for N days ago
   - Parameters: `$1` = days back
   - Uses `get_platform()`:
     - darwin: `date -u -v-"${1}d" +%s`
     - linux: `date -u -d "${1} days ago" +%s`
   - Echo epoch timestamp

4. **Helper: `_normalize_model()`** -- Extract model family from full identifier
   - Parameters: `$1` = full model string (e.g., "claude-opus-4-6")
   - Use case statement:
     - `*opus*`) echo "opus" ;;
     - `*sonnet*`) echo "sonnet" ;;
     - `*haiku*`) echo "haiku" ;;
     - `*`) echo "other" ;;

5. **Helper: `_iso_to_epoch()`** -- Convert ISO 8601 timestamp to epoch
   - Parameters: `$1` = ISO timestamp (e.g., "2026-03-06T14:49:00.971Z")
   - Strip milliseconds and Z: `local clean=$(echo "$1" | sed 's/\.[0-9]*Z$//' | sed 's/Z$//')`
   - Cross-platform:
     - darwin: `date -u -j -f "%Y-%m-%dT%H:%M:%S" "$clean" +%s 2>/dev/null`
     - linux: `date -u -d "${clean}" +%s 2>/dev/null`
   - Return 0 on failure (skip unparseable timestamps)

6. **Function: `_find_session_files()`** -- Find all JSONL session files
   - Glob pattern: `$HOME/.claude/projects/*/*.jsonl`
   - IMPORTANT: No `sessions/` subdirectory -- files are directly in project dirs
   - Also check subagent dirs: `$HOME/.claude/projects/*/*/subagents/*.jsonl`
   - Filter out files older than 30 days (by mtime) to limit scan scope using `get_file_mtime()`
   - Output file paths, one per line
   - Return 1 if no files found

7. **Function: `parse_local_logs()`** -- Main parser, aggregates token usage
   - Get plan type: `local plan_type=$(get_tmux_option "@claudux_plan" "max_5x")`
   - Get limits: `read weekly_limit monthly_limit <<< "$(_get_plan_limits "$plan_type")"`
   - Get cutoff epochs: `local cutoff_7d=$(_get_cutoff_epoch 7)` and `local cutoff_30d=$(_get_cutoff_epoch 30)`
   - Find session files: `_find_session_files` -- if returns 1, output error JSON and return
   - Initialize accumulators (use temp files or variables):
     - `total_tokens_7d=0`, `total_tokens_30d=0`
     - `sonnet_tokens=0`, `opus_tokens=0`, `haiku_tokens=0`
   - For each JSONL file, process line by line:
     ```bash
     while IFS= read -r line; do
         # Use jq to extract type, filtering early
         local entry_type
         entry_type=$(printf '%s' "$line" | jq -r '.type // empty' 2>/dev/null) || continue
         [[ "$entry_type" != "assistant" ]] && continue

         # Extract fields
         local model input_tokens output_tokens timestamp_str
         model=$(printf '%s' "$line" | jq -r '.message.model // empty' 2>/dev/null) || continue
         [[ -z "$model" ]] && continue
         input_tokens=$(printf '%s' "$line" | jq -r '.message.usage.input_tokens // 0' 2>/dev/null)
         output_tokens=$(printf '%s' "$line" | jq -r '.message.usage.output_tokens // 0' 2>/dev/null)
         timestamp_str=$(printf '%s' "$line" | jq -r '.timestamp // empty' 2>/dev/null)
         [[ -z "$timestamp_str" ]] && continue

         local entry_epoch
         entry_epoch=$(_iso_to_epoch "$timestamp_str") || continue
         [[ "$entry_epoch" -eq 0 ]] && continue

         local tokens=$(( input_tokens + output_tokens ))
         local normalized_model
         normalized_model=$(_normalize_model "$model")

         # Aggregate by time window
         if [[ "$entry_epoch" -ge "$cutoff_30d" ]]; then
             total_tokens_30d=$(( total_tokens_30d + tokens ))
             # Per-model accumulation
             case "$normalized_model" in
                 sonnet) sonnet_tokens=$(( sonnet_tokens + tokens )) ;;
                 opus)   opus_tokens=$(( opus_tokens + tokens )) ;;
                 haiku)  haiku_tokens=$(( haiku_tokens + tokens )) ;;
             esac
         fi
         if [[ "$entry_epoch" -ge "$cutoff_7d" ]]; then
             total_tokens_7d=$(( total_tokens_7d + tokens ))
         fi
     done < "$file"
     ```
   - PERFORMANCE NOTE: jq per line is slow for large files. An optimization would be to use a single jq pass per file:
     `jq -r 'select(.type == "assistant") | [.timestamp, .message.model, (.message.usage.input_tokens // 0), (.message.usage.output_tokens // 0)] | @tsv'`
     Then process TSV output in bash. Use this optimized approach.
   - Try to find account email from `~/.claude/settings.json` or `~/.claude/.credentials.json`:
     ```bash
     local email="local"
     if [[ -f "$HOME/.claude/settings.json" ]]; then
         email=$(jq -r '.email // "local"' "$HOME/.claude/settings.json" 2>/dev/null)
     fi
     ```
   - Calculate percentages: `weekly_pct=$(echo "scale=1; $total_tokens_7d * 100 / $weekly_limit" | bc 2>/dev/null || echo "0")`
   - Build normalized JSON matching cache schema:
     ```json
     {
       "mode": "local",
       "fetched_at": <epoch>,
       "account": {"email": "<email>"},
       "weekly": {"used": <weekly_pct>, "limit": 100, "unit": "percent", "reset_at": 0},
       "monthly": {"used": <monthly_pct>, "limit": 100, "unit": "percent", "reset_at": 0},
       "models": {
         "sonnet": {"used": <sonnet_pct>, "limit": 100, "unit": "percent", "reset_at": 0},
         "opus": {"used": <opus_pct>, "limit": 100, "unit": "percent", "reset_at": 0}
       },
       "error": null
     }
     ```
   - For model percentages, use each model's share of the weekly limit (simple: use weekly_limit for each model too since we don't have per-model limits)
   - On no files found, output error: `{"mode":"local","fetched_at":<epoch>,...,"error":{"code":"no_logs","message":"No Claude Code session logs found at ~/.claude/projects/"}}`
   - Echo the final JSON to stdout
