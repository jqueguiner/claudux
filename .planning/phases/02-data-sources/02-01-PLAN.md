<task type="auto">
  <name>Task 1: Create Admin API fetch functions</name>
  <files>scripts/api_fetch.sh</files>
  <action>
Create `scripts/api_fetch.sh` with the following structure:

1. **Header and sourcing:**
   - Shebang `#!/usr/bin/env bash`
   - Comment header: "api_fetch.sh -- Anthropic Admin API client for claudux"
   - Source helpers.sh: `CURRENT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` then `source "$CURRENT_DIR/helpers.sh"`
   - Define `CLAUDUX_API_BASE="https://api.anthropic.com"`

2. **Helper: `_api_request()`** -- Internal function for all API calls
   - Parameters: `$1` = URL path (e.g., `/v1/organizations/usage_report/messages`), `$2` = query string, `$3` = API key
   - Use `curl -s -w "\n%{http_code}" --connect-timeout 10 --max-time 30` with headers:
     - `-H "anthropic-version: 2023-06-01"`
     - `-H "x-api-key: $3"`
   - Split response: `http_code=$(echo "$response" | tail -1)`, `body=$(echo "$response" | sed '$d')`
   - Error handling by HTTP status:
     - 200: echo "$body" and return 0
     - 401|403: echo JSON error `{"code":"auth_failed","message":"Invalid or insufficient API key"}` to stderr, return 1
     - 429: echo JSON error `{"code":"rate_limited","message":"Rate limited -- increase refresh interval"}` to stderr, return 1
     - 5*): echo JSON error `{"code":"api_unavailable","message":"Anthropic API unavailable (HTTP $http_code)"}` to stderr, return 1
     - *): echo JSON error `{"code":"unknown","message":"Unexpected HTTP $http_code"}` to stderr, return 1

3. **Helper: `_get_date_range()`** -- Cross-platform date calculation
   - Parameters: `$1` = days back (e.g., 7 or 30)
   - Uses `get_platform()` to branch:
     - darwin: `date -u -v-"${1}d" +"%Y-%m-%dT00:00:00Z"`
     - linux: `date -u -d "${1} days ago" +"%Y-%m-%dT00:00:00Z"`
   - Also output end date: `date -u +"%Y-%m-%dT23:59:59Z"`
   - Echo both as space-separated: `echo "$start $end"`

4. **Function: `fetch_usage_report()`** -- Fetch token usage from Admin API
   - Parameters: `$1` = API key, `$2` = days back (default: 7)
   - Build URL: `${CLAUDUX_API_BASE}/v1/organizations/usage_report/messages`
   - Query params: `starting_at=${start}&ending_at=${end}&group_by[]=model&bucket_width=1d`
   - Handle pagination loop: call `_api_request`, check `has_more` with jq, follow `next_page`
   - Accumulate all `.data[]` entries into a single JSON array
   - Output merged data array to stdout

5. **Function: `fetch_cost_report()`** -- Fetch cost data
   - Parameters: `$1` = API key, `$2` = days back (default: 7)
   - URL: `${CLAUDUX_API_BASE}/v1/organizations/cost_report`
   - Query: `starting_at=${start}&ending_at=${end}&bucket_width=1d`
   - Handle pagination same as usage report
   - Output data array to stdout

6. **Function: `fetch_org_info()`** -- Get organization name and first user email
   - Parameters: `$1` = API key
   - Call `/v1/organizations/me` to get org name
   - Call `/v1/organizations/users?limit=1` to get first member email
   - Output JSON: `{"org_name": "...", "email": "..."}`

7. **Function: `api_fetch()`** -- Main entry point, produces normalized cache JSON
   - Parameters: `$1` = API key
   - Call `fetch_usage_report "$1" 7` for weekly data
   - Call `fetch_usage_report "$1" 30` for monthly data
   - Call `fetch_cost_report "$1" 7` for cost data
   - Call `fetch_org_info "$1"` for account info
   - Normalize model names: extract family (opus/sonnet/haiku) from full model strings using case pattern matching (`*opus*`, `*sonnet*`, `*haiku*`)
   - Sum tokens per model across all time buckets using jq:
     - For each model: sum `uncached_input_tokens + cache_read_input_tokens + output_tokens`
   - Build normalized JSON matching the cache schema from CONTEXT.md:
     ```json
     {
       "mode": "org",
       "fetched_at": <epoch>,
       "account": {"email": "..."},
       "weekly": {"used": <total_tokens_7d>, "limit": 0, "unit": "tokens", "reset_at": 0},
       "monthly": {"used": <total_tokens_30d>, "limit": 0, "unit": "tokens", "reset_at": 0},
       "models": {
         "sonnet": {"used": <sonnet_tokens>, "limit": 0, "unit": "tokens", "reset_at": 0},
         "opus": {"used": <opus_tokens>, "limit": 0, "unit": "tokens", "reset_at": 0}
       },
       "error": null
     }
     ```
   - Note: `limit` is 0 for org mode because the API doesn't expose tier limits. Phase 3 will handle display accordingly. `reset_at` is 0 for now (billing cycle unknown).
   - On any fetch error, produce error JSON: `{"mode":"org","fetched_at":<epoch>,...,"error":{"code":"...","message":"..."}}`
   - Echo the final JSON to stdout (caller is responsible for writing to cache)

8. **Make executable:** `chmod +x scripts/api_fetch.sh`

IMPORTANT: This script does NOT write to cache directly. It outputs JSON to stdout. The `fetch.sh` orchestrator (Plan 02-03) will call this and pipe to `cache_write()`.
