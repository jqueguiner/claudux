  <action>
Create `scripts/cache.sh` with these functions. Source helpers.sh at the top.

1. `cache_read()` — Reads and outputs the contents of the cache file (`$(get_cache_dir)/cache.json`). Returns 1 if file doesn't exist. No network calls — this is what status bar scripts call.

2. `cache_write(json_content)` — Atomically writes content to cache file:
   - Get cache dir via `get_cache_dir`
   - Create tmpfile with `mktemp "${cache_dir}/cache.XXXXXX"`
   - Write content to tmpfile via `printf '%s\n' "$1" > "$tmpfile"`
   - Atomic rename: `mv -f "$tmpfile" "${cache_dir}/cache.json"`
   - Trap to clean up tmpfile on error: `trap 'rm -f "$tmpfile"' ERR`
   - Return 0 on success, 1 on failure

3. `is_cache_stale(ttl_seconds)` — Checks if cache is older than TTL:
   - If cache file doesn't exist, return 0 (stale)
   - Get mtime via `get_file_mtime`
   - Get current epoch via `date +%s`
   - Compare: `(now - mtime) >= ttl` → return 0 (stale), else return 1 (fresh)
   - If no ttl_seconds provided, read from tmux option `@claudux_refresh_interval` with default from `$CLAUDUX_DEFAULT_REFRESH_INTERVAL`

4. `acquire_lock()` — Cross-platform file locking:
   - Lock path: `$(get_cache_dir)/cache.lock`
   - On Linux (flock available): `exec 9>"$lockpath" && flock -w 10 9`
   - On macOS (no flock): mkdir-based lock at `${lockpath}.d`
     - Try `mkdir "$lockdir"` in a loop with 0.1s sleep
     - Timeout after 10 seconds
     - Write PID to `"$lockdir/pid"` for stale detection
     - Before failing on timeout, check if stored PID is alive (`kill -0`), force-remove if dead
   - Return 0 on success, 1 on timeout

5. `release_lock()` — Release the lock:
   - If flock: `exec 9>&-`
   - If mkdir: `rm -rf "${lockpath}.d"`

Create `scripts/credentials.sh` with these functions. Source helpers.sh at the top.

1. `load_api_key()` — Load API key with this priority:
   - Check `$ANTHROPIC_ADMIN_API_KEY` env var first. If non-empty, echo it and return 0.
   - Check config file at `$(get_config_dir)/credentials`. If file exists:
     - Verify permissions are 600 using platform-aware stat:
       - Linux: `stat -c %a "$config_file"`
       - macOS: `stat -f %Lp "$config_file"`
     - If permissions are NOT 600, print warning to stderr: "claudux: $config_file has insecure permissions ($perms). Expected 600." and return 1.
     - Read first non-empty, non-comment line: `grep -v '^[[:space:]]*#' "$config_file" | grep -v '^[[:space:]]*$' | head -1`
     - Echo it and return 0.
   - If nothing found, return 1 (no key available)

2. `get_key_type(key)` — Detect key type:
   - If key starts with `sk-ant-admin`: echo "admin" (org mode)
   - Otherwise: echo "unknown"
   - Return 0

IMPORTANT: Neither script should ever pass the API key as a command-line argument. The key flows through environment variables, file reads, and shell variable assignment only.

Both scripts should be executable (`chmod +x`) and use `#!/usr/bin/env bash` shebang.
