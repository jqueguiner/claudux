<task type="auto">
  <name>Task 1: Create config/defaults.sh and scripts/helpers.sh</name>
  <files>config/defaults.sh, scripts/helpers.sh</files>
  <action>
Create `config/defaults.sh` with all default tmux option values. This file is sourced by helpers.sh and provides the single source of truth for defaults:

```bash
#!/usr/bin/env bash
# Default values for all @claudux_* tmux options
# Sourced by helpers.sh — never executed directly

CLAUDUX_DEFAULT_REFRESH_INTERVAL=300
CLAUDUX_DEFAULT_BAR_LENGTH=10
CLAUDUX_DEFAULT_WARNING_THRESHOLD=50
CLAUDUX_DEFAULT_CRITICAL_THRESHOLD=80
CLAUDUX_DEFAULT_SHOW_WEEKLY="on"
CLAUDUX_DEFAULT_SHOW_MONTHLY="on"
CLAUDUX_DEFAULT_SHOW_MODEL="on"
CLAUDUX_DEFAULT_SHOW_RESET="on"
CLAUDUX_DEFAULT_SHOW_EMAIL="off"
```

Create `scripts/helpers.sh` with these functions:

1. `get_tmux_option(option, default_value)` — reads tmux global option with `tmux show-option -gqv "$option"`, returns default if empty. Follow the exact tmux-battery pattern.

2. `set_tmux_option(option, value)` — sets tmux global option with `tmux set-option -gq "$1" "$2"`.

3. `get_platform()` — returns "linux" or "darwin" based on `uname -s`. Cache the result in a variable to avoid repeated uname calls.

4. `get_file_mtime(filepath)` — returns epoch mtime. Linux: `stat -c %Y "$file"`. macOS: `stat -f %m "$file"`. Uses get_platform for dispatch.

5. `get_cache_dir()` — returns `${XDG_CACHE_HOME:-$HOME/.cache}/claudux`, creating it with `mkdir -p` if missing.

6. `get_config_dir()` — returns `${XDG_CONFIG_HOME:-$HOME/.config}/claudux`.

7. `get_plugin_dir()` — returns the directory containing the claudux.tmux file. Use `CURRENT_DIR` pattern from tmux-battery: detect via `${BASH_SOURCE[0]}` relative path.

Use `#!/usr/bin/env bash` shebang. Source config/defaults.sh at the top of helpers.sh using the plugin directory for path resolution. All function names should use snake_case.

Key implementation notes:
- Platform result should be cached in a module-level variable (detect once, reuse)
- `get_tmux_option` must handle the case where tmux is not running (return default gracefully)
- Do NOT use associative arrays (Bash 4+ feature that complicates things unnecessarily here)

<task type="auto">
  <name>Task 2: Create scripts/check_deps.sh</name>
  <files>scripts/check_deps.sh</files>
  <action>
Create `scripts/check_deps.sh` — a standalone script that checks for required dependencies and warns via tmux display-message if anything is missing. This script is called from claudux.tmux at plugin load time.

Source helpers.sh at the top for utility functions.

Check these dependencies:
1. **Bash version**: `${BASH_VERSINFO[0]}` must be >= 4. If too old, warn: "claudux: Bash 4.0+ required (found $BASH_VERSION). macOS: brew install bash"
2. **jq**: `command -v jq`. If missing, warn: "claudux: jq required. Install: brew install jq / sudo apt install jq"
3. **curl**: `command -v curl`. If missing, warn: "claudux: curl required. Install: brew install curl / sudo apt install curl"

Display warnings via `tmux display-message` with a 2>/dev/null guard (in case tmux is not fully initialized). Do NOT exit or crash — just warn and return a non-zero exit code if any dependency is missing. This keeps tmux functional even if claudux can't run.

The script should be executable (`chmod +x`).

Make the script both sourceable and directly executable. When executed directly, run the check function and exit with appropriate code. When sourced, just define the function.
