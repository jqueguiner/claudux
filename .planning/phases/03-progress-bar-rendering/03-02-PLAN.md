  <files>scripts/render.sh</files>
  <action>
Append two new functions to the existing `scripts/render.sh`:

**render_model_sonnet function:**
- Same pattern as render_weekly/render_monthly
- Reads cache once: `cache_data=$(cache_read) || return 0`
- Checks for error: `printf '%s' "$cache_data" | jq -e '.error == null' >/dev/null 2>&1 || return 0`
- Extracts `used` and `limit` from `.models.sonnet` via jq
- Guard: If `.models.sonnet` doesn't exist in cache JSON, return 0 (silent)
- Guard: If `limit` is 0, return 0 (silent — org mode doesn't have model limits)
- Calculates percentage, caps at 100
- Reads bar_length from tmux option
- Outputs: `#[fg=colour245]S:#[default] ` + `$(render_bar "$pct" "$bar_length")` + ` ${pct}%`
- Label `S:` uses same `#[fg=colour245]` dim gray as W: and M:

**render_model_opus function:**
- Identical pattern to render_model_sonnet
- Reads from `.models.opus` instead
- Label: `O:`

**Important edge cases:**
- The `models` object may be empty `{}` (org mode with no model data). `jq -r '.models.sonnet.used // 0'` handles this safely.
- The `models` key may not contain `sonnet` or `opus` at all. Check with `jq -e '.models.sonnet'` before extracting.
