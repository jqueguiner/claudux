Create `scripts/render.sh` with the standard source chain pattern and the core `render_bar` function.

**File header:**
```bash
#!/usr/bin/env bash
# render.sh — Progress bar rendering for claudux tmux plugin
# Produces tmux-formatted strings with Unicode progress bars and color coding.
# Each render_* function outputs a self-contained segment for the status bar.

CURRENT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "$CURRENT_DIR/helpers.sh"
source "$CURRENT_DIR/cache.sh"
```

**render_bar function:**
- Parameters: `$1` = percentage (integer 0-100), `$2` = bar_length (default 10)
- Reads thresholds via `get_tmux_option "@claudux_warning_threshold"` and `"@claudux_critical_threshold"` with defaults from `defaults.sh`
- Color logic:
  - `pct >= critical_threshold` -> `colour196` (red)
  - `pct >= warning_threshold` -> `colour220` (yellow)
  - else -> `colour34` (green)
- Calculates filled segments: `$(( (pct * bar_length + 50) / 100 ))` (round to nearest integer)
- Builds bar string using a loop: `[#[fg=colourN]` + filled `█` chars + `#[default]` + empty `░` chars + `]`
- Uses `printf '%s'` to output (no trailing newline)
- Important: Read thresholds ONCE at function start, not inside loops. Each `get_tmux_option` call forks a process.

**render_weekly function:**
- Reads cache once: `cache_data=$(cache_read) || return 0`
- Checks for error in cache: `printf '%s' "$cache_data" | jq -e '.error == null' >/dev/null 2>&1 || return 0`
- Extracts `used` and `limit` from `.weekly` via jq
- If `limit` is 0 or empty, return 0 (silent — no bar when limit unknown)
- Calculates percentage: `pct=$(( (used * 100) / limit ))`, cap at 100
- Reads bar_length from tmux option
- Outputs: `#[fg=colour245]W:#[default] ` + `$(render_bar "$pct" "$bar_length")` + ` ${pct}%`
- Label uses `#[fg=colour245]` (medium gray) for subtle dimming per research recommendation

**render_monthly function:**
- Same pattern as render_weekly but reads from `.monthly`
- Label: `M:`

Make the file executable: `chmod +x scripts/render.sh`
