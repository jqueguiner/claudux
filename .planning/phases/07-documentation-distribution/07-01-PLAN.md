3. **Requirements**: List prerequisites from check_deps.sh:
   - tmux 3.0+
   - Bash 4.0+ (macOS: `brew install bash`)
   - jq (`brew install jq` / `sudo apt install jq`)
   - curl (`brew install curl` / `sudo apt install curl`)

4. **Installation**: Two subsections:
   - **TPM (recommended)**: Add `set -g @plugin 'user/claudux'` to ~/.tmux.conf, then `prefix + I`
   - **Manual**: `git clone https://github.com/user/claudux ~/.tmux/plugins/claudux`, add `run-shell ~/.tmux/plugins/claudux/claudux.tmux` to ~/.tmux.conf, then `tmux source ~/.tmux.conf`
   - Use placeholder `user/claudux` for the GitHub org

5. **Usage**: Show how to add format strings to status-right. Example:
   ```
   set -g status-right '#{claudux_weekly} #{claudux_sonnet} #{claudux_opus} #{claudux_reset}'
   ```

6. **Format Strings**: Table extracted from claudux.tmux `claudux_interpolation` array:
   | Format String | Renders |
   |`#{claudux_weekly}` | Weekly quota progress bar |
   |`#{claudux_monthly}` | Monthly quota progress bar |
   |`#{claudux_sonnet}` | Sonnet model usage bar |
   |`#{claudux_opus}` | Opus model usage bar |
   |`#{claudux_reset}` | Reset countdown (e.g., 2h 15m) |
   |`#{claudux_email}` | Account email |
   |`#{claudux_status}` | Error/stale indicator |

7. **Configuration**: Table extracted from config/defaults.sh defaults:
   | Option | Default | Description |
   |`@claudux_show_weekly` | `on` | Show weekly quota bar |
   |`@claudux_show_monthly` | `on` | Show monthly quota bar |
   |`@claudux_show_model` | `on` | Show Sonnet/Opus bars |
   |`@claudux_show_reset` | `on` | Show reset countdown |
   |`@claudux_show_email` | `off` | Show account email |
   |`@claudux_bar_length` | `10` | Progress bar width (5-30) |
   |`@claudux_warning_threshold` | `50` | Yellow threshold (%) |
   |`@claudux_critical_threshold` | `80` | Red threshold (%) |
   |`@claudux_refresh_interval` | `300` | Cache TTL in seconds |

   Include an example tmux.conf snippet showing customization.

8. **Data Sources**: Two subsections:
   - **Org Mode (API users)**: Explain Admin API key requirement. Steps: go to Anthropic Console -> Organization Settings -> Admin API keys -> Create key. Set via `export ANTHROPIC_ADMIN_API_KEY=sk-ant-admin...` or config file at `~/.config/claudux/credentials` (must be chmod 600). Key must start with `sk-ant-admin`. Shows monthly spend, per-model usage.
   - **Local Mode (Claude Code subscribers)**: No API key needed. Auto-detected when no admin key present and `~/.claude/projects/` contains JSONL session logs. Shows estimated usage from local log parsing. Note: may not match exact Anthropic dashboard numbers.

9. **Troubleshooting**: Cover these exact scenarios:
   - "Nothing shows up" -> check `tmux source ~/.tmux.conf`, verify deps (jq, curl, bash 4+)
   - "Shows [!] auth_failed" -> verify Admin API key is correct and starts with `sk-ant-admin`
   - "Shows stale ?" -> check internet connectivity, try increasing @claudux_refresh_interval
   - "Bars show 0%" -> in local mode, check `~/.claude/projects/` exists with .jsonl files

10. **License**: "MIT License - see [LICENSE](LICENSE)"

**Tone:** Professional, scannable, concise. No walls of text. Use headers, tables, and code blocks throughout. Do NOT add badges, emojis, or a Contributing section for v1.
